<article>

Se utiliza para especificar contenido independiente y autocontenido. Este debe tener sentido por sí mismo y debe ser posible distribuirlo independientemente del resto del sitio.

Es soportado por los siguientes navegadores:

  • Google Chrome 6.0
  • Edge 9.0
  • Firefox 4.0
  • Safari 5.0
  • Opera 11.1.

Admite los atributos generales de HTML, los cuales son:

  • contenteditable: especifica si el contenido de un elemento es editable o no.
  • data-*: usado para almacenar datos privados en la página o aplicación.
  • draggable: especifica si el elemento es arrastrable o no.
  • dropzone: especifica si los datos arrastrados se copian, mueven o vinculan cuando se sueltan.
  • hidden: especifica que un elemento es, o si no es pertinente, relevante.
  • spellcheck: especifica si el elemento debe tener su ortografía y gramática revisadas o no.
  • translate: especifica si el contenido de un elemento debe ser traducido o no.

Admite los atributos de evento:

  • onafterprint: secuencia de comandos a ejecutar después de que se imprime el documento.
  • onbeforeprint: secuencia de comandos a ejecutar antes de que se imprima el documento.
  • onbeforeunload: secuencia de comandos a ejecutar cuando el documento está a punto de descargarse.
  • onerror: secuencia de comandos a ejecutar cuando un error ocurre.
  • onhashchange: secuencia de comandos a ejecutar cuando ha habido cambios a la parte de anclaje de la URL.
  • onmessage: secuencia de comandos a ejecutarse cuando un mensaje es activado.
  • onoffline: secuencia de comandos a ejecutarse cuando el buscador comienza a trabajar fuera de línea.
  • ononline: secuencia de comandos a ejecutar cuando el buscador comienza a trabajar en línea.
  • onpagehide: secuencia de comandos a ejecutar cuando un usuario navega fuera de una pagina.
  • onpageshow: secuencia de comandos a ejecutar cuando un usuario navega a una página.
  • onpopstate: secuencia de comandos a ejecutar cuando el historial de la ventana cambia.
  • onresize: Se desencadena cuando cambia el tamaño de la ventana del navegador.
  • onstorage: secuencia de comandos a ejecutar cuando se actualiza un área de almacenamiento web.

Ejemplo de código:

<article> <h1> Unicornio </h1> <p> Es una criatura mitológica.</p> </article>
